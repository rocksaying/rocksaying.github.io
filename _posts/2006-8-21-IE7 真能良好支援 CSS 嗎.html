微軟員工鬆口承認 IE6 有一卡車的錯誤，並浪費了程式人員難以計數的時間。見：IE7 and standards compliance - Microsoft's Chris Wilson charts progress。這下我可以理直氣壯的向微軟怒吼「還我加班時間!!」

在該文中，微軟員工提到 IE7 承諾會提供 W3C CSS 2.1 的完整支援。照以往微軟的承諾來看，我實在是不樂觀。同時也提到， IE7 目前的開發工作中，保證舊版相容性 (backwards compatibility) 是更重要的工作項目。我說啊，這些問題不就是微軟自己搞出來的嗎？當人們為了閃避 IE6 的 bug ，各顯神通地寫出一堆怪模怪樣的網頁語法時，微軟不問不聞，了不得在 MSDN 裡面發佈一些「解決小技巧」。例如，A bug about radio input and createElement in M$IE提的例子。現在微軟自己要更新 IE 了，才發現那累積了四年的各式古怪語法，竟然變成了「重要的工作項目」，真是令人啼等皆非。
