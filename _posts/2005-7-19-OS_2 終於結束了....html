剛在 Taiwan CNet 看到 這篇文章 IBM向OS/2告別 ，心中真是感觸萬千。如果 96-98 年時， IBM 就把 OS/2 的 source 開放出來，也不至於搞到這樣。至少當年我在玩 OS/2 3.0T/4.0T 時，它在 i18n/l10n 上的腳步領先當年的 linux (當年要在 linux 上跑中文，是非常痛苦的一件事) ，也比 windows 95 好些。

再仔細想想，我也是在 OS/2 摸了好一陣子多工系統的 api 概念後，才轉到 linux 下。那時我還把原本跑在 unix 系統下的 Firebird BBS 移植到 OS/2 上跑，當時的 codes ，現在還能在我的一些 sources (FBTIP) 中看到。

從核心程式來看，OS/2 在那時候就很成熟與穩定了，同時的 Windows NT 4.0 還在苦苦開拓市場。 NT? (New Technology) 裡一些東西，在 OS/2 2.x 中就已經在運作了，當微軟決定放棄 OS/2 ，全力拱起 NT 時，想必是借助了不少 OS/2 的開發經驗 (老世代的玩家應該還記得 OS/2 1.x,2.x 是 IBM 和 微軟共同開發的)。如果那時就引進 FS/OSS 的概念，把 OS/2 的 source 開放出來，說不定現在 FS/OSS 裡的 OS 架構就會有兩大門派， application 也會互通。

可惜的是，現在說這些太遲了。我只能說，這是一個電腦作業系統世代的句號。

突然想起自己還把 OS/2 3.0T,4.0T 的光碟留著，不知還能不用能。 瞻仰一下吧 XD
