八月十日，看到有人提供瑞麗發佈新版 xfire 1723 驅動程式的消息...

八月十日，看到有人提供瑞麗發佈新版 xfire 1723 驅動程式的消息，馬上去看瑞麗的官網。發現瑞麗這次自己做了兩塊 xfire 1723 的卡，叫做 Sound Image 火網1723 / 火網Mini 。看了一下規格，跟 TerraTec Aureon Xfire 1723 有一個地方明顯不同，那就是數位輸出的規格，竟然只到 48KHz ，我當場就虛了。至於 Sound Image 火網 1723 上的類比輸出 (20bit 的數位類比轉換線路) 是否比 TerraTec Aureon Xfire 1723 好，我沒聽過就不清楚了。

Sound Image 火網1723 / 火網Mini 驅動程式下載頁: http://www.mediatek.com.tw/big5/support/driver/index.htm

我用的是 TerraTec Aureon Xfire 1723 ，到上面的下載頁，抓了瑞麗發佈的驅動程式來試用。看那安裝畫面跟 VIA 官版驅動程式一樣... 不過安裝之後，那個控制介面是真的比之前好看多了，還多了個「效能模式」的選項 (我現在還搞不清楚它的功用)。

裝了之後，先試類比輸出 (foobar: playbak with 32bit fixed-point, output by KS, 不掛任何 DSP; 輸出規格 44.1KHz) ，正常，但聽不出來驅動程式安裝後的音質有何改善。再試數位輸出，先按瑞麗的規格設定 48KHz (foobar: playback with 32bit fixed-point, output by KS, 掛 Resampler 設定 48KHz) ，正常輸出。再按 TerraTec 的規格設定 96KHz (foobar: Resampler 設定 96KHz, 其餘同前) ，這可抱歉了， foobar 跳出錯誤訊息，無法開啟音效設備。

OK, 試到這我就知道了，用 TerraTec Aureon Xfire 1723 接 DAC 的使用者，繼續用 TerraTec 光碟附的驅動程式就好了。瑞麗的驅動程式跟 VIA 官版的一樣，數位輸出都只支援到 48KHz 而已。

不過瑞麗的控制面板很漂亮，我想留著繼續用。所以我進到系統的「控制台 \ 裝置管理員」，找到 Envy24 Family Audio Controller WDM 裝置後，用「更新驅動程式」功能，強制把驅動程式換回 TerraTec 的版本。更新過程中，會跳出「已安裝的 Envy24HF.sys 比選擇的版本新」的訊息，選替換就對了。另外還會提示幾個 .dll 要不要替換，這就不用替換了。只有 Envy24HF.sys 要換回 TerraTec 的版本。「更新??」後重開機。

重開機後看了看，嘿嘿，果然還是那個漂亮的控制面板，設定功能都正常，只少了一個「Immerzio 3D Effect」的選項，反正我從不用那東西，無妨。驅動程式的版本也顯示是 TerraTec 的號碼。再試著用 foobar 以 96KHz 試數位輸出功能。成功輸出啦。哈哈。

TerraTec 的版本號碼: 5.12.1.3632; 瑞麗的版本則是 5.12.1.3647

以 96KHz 試驗數位輸出的圖: 注意看 Current 是 96KHz ，用瑞麗版本的驅動程式可跑不出這個數。

我不知道瑞麗的 Sound Image 火網 1723 能不能這樣玩，有試出來的仁兄說說意見吧。

其實依 xfire 1723 的硬體設計，在一般情形下，類比跟數位輸出端子都會有訊號。也就是喇叭不管接哪個端子，都會發聲。

不過，類比輸出端子最高只能處理 48KHz 的訊號，所以當用 foobar resample 超過48KHz 時，就只有數位輸出端子有訊號了。如果你沒有 DAC (喇叭只能接在類比輸出端子) ，這時你就會發現 foobar 明明在播放，可是你的喇叭一點聲音都沒有。

你檢查一下:

1.用舊版介面時有沒有問題？
如果用舊版介面沒問題，那就整個換回舊版吧。畢竟新介面 + 舊 driver 不是正式的用法，會有相容性問題。

2.檢查線材。

3.先試著用 44.1khz/16bit 格式輸出。
foobar output 16bit, 不用 Resampler 。因為舊型的 DAC ，只接受這格式的數位訊號。

4.再試著用 96khz/24bit 格式輸出。
用 CS8414 做數位訊號接收晶片的機子，可以接收這格式。

5.最後試 88.2khz/32bit 格式輸出。
88.2khz/32bit 剛好是 CD 格式 (44.1khz/16bit) 訊號的線性放大兩倍。技術上來說，線性放大的 re-sample 波形，跟原音一樣，所以比 96khz 好些。但有些 DAC 能接受的數位訊號格式有限，不接受 88.2khz 格式。

你指的是 TerraTec Aureon X-Fire 1723 的官版驅動程式吧？ 我剛好前幾天裝起來測過，結論是新不如舊。

首先，它不像舊版的那樣，可以「原採樣率」輸出。在舊版中，當你選擇「Sample Rate Auto select」時，你輸出的訊號採樣率是多少，它就輸出多少。若訊號是 88.2khz 就輸出 88.2khz 。可是新版不能如此了。新版只接受 44.1khz, 48khz 以及 96khz 三種採樣率。當我在 foobar2k 中試圖輸出 88.2khz 採樣率的訊號時， foobar2k 就會顯示音效裝置不支援此種訊號。

而且新版中的 Hi Sample Rate 模式 (96khz) 模式下，似乎會把 48khz 以下的訊號採樣率自動提昇到 96khz 。再者，在此模式下，聲音聽來尖了不少，感覺並不理想。

ps. 我使用的 DAC 為 電光火石 Spitfire DAC 。

wshaowei ，你說的狀況是正常的。當我們使用 Kernel Streaming 輸出時，如果有其他軟體也要使用同一個音效設備播音訊，就會產生爭搶資源的狀況，中斷 Kernel Streaming 的輸出動作。原因好像在於 Kernel streaming 沒有混音的能力 (因為它認為自己獨佔了音效設備)。

DirectSound 則有混音的能力，當同時有多個軟體透過同一個音效設備輸出時，DirectSound 會把它們輸出的音訊混在一起。但這個動作，會改變音響。對重視 Hi-end 高傳真的發燒友來說，這是不能接受的缺點。

不論我用 DS, KS 或 ASIO ，我都是用數位輸出到 spitfire DAC 。我目前主要是用 foobar2000 + ASIO + xfire 1723 。我個人聽感，KS和ASIO的音質不分高下，但音樂走向有極微小的差異。

此外，雖然 ASIO 的設定比較複雜，但好處是，不怕被其他軟體打斷foobar2k的輸出。

xfire1723 不需要開啟 SSRC ，搭配數位輸出時更不建議開啟 SSRC ，音質反而會降低。output format 可以選 32bit。

ASIO 的設定很簡單，延遲(delay)和緩衝(buffer)愈低愈好，最好是0；不要開啟resample。

聲音的差異，雖說可能是個人偏好，但基本上，ASIO的音質比較接近原音。
